# Origin v1.5.0 Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Merge upstream `origin/main` (`v1.5.0`) into the locally customized project without losing local proxy-management features, low-memory Docker build settings, or deployment-specific defaults.

**Architecture:** Perform a real three-way merge on a dedicated integration branch using `git merge --no-ff --no-commit origin/main`, then resolve conflicts in layers: backend/shared core first, frontend/routes second, local deployment/build customizations third. Validation must run serially after merge resolution so embedded frontend assets exist before Go package tests execute.

**Tech Stack:** Git, Go 1.25, React 19, TypeScript, Vite, SQLite/Postgres migrations, wsrelay/auth/proxy backend modules, Docker multi-stage build
